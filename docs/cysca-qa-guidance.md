# Cysca QA Guidance

This repository includes a Codex skill for QA work on Cysca complex systems.

## Skill source

- `skills/cysca-qa-complex-systems/SKILL.md`
- `skills/cysca-qa-complex-systems/references/cysca-qa-notes.md`

## Install locally

```bash
./scripts/install-cysca-qa-skill.sh
```

## What the skill is for

- risk-based QA for complex systems
- test strategy and traceability
- defect analysis and reporting
- automation prioritization

## Why it matters here

The repo demonstrates how automation can be framed around a real QA mindset:

- focus on system behavior, not isolated steps
- document risks, dependencies, and recovery behavior
- keep evidence and traceability easy for reviewers to follow
